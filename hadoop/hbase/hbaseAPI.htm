hbase API

  • SCAN:扫描器
    缓存:旨在减少client向server端的RPC次数
    行缓存caching:一个KeyValue为一行
    列缓存batch:相同rowkey，family的列数
    如：
    rowkey family/column value
    1           cf1/t1             1
    2           cf1/t1             2
    2           cf1/t2             2
    2           cf1/t3             2
    3           cf1/t1             3
    caching batch rpc
    1           1       5+(1)
    2           1       3+(1)
    1           2       4+(1)
    2           2       2+(1)
    1           3       3+(1)
    2           3       2+(1)
    3           3       1+(1)
    (1)为确认扫描完成的RPC
